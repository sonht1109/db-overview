Chapter 28. Case Studies

Chapter 28 Summary: What We Learned

Page Topic Takeaway in One Sentence
271 A to-do app database Simple schema: users, tasks, lists. Teaches basic normalization, indexes on foreign keys, and common query patterns.
272 An e-commerce database Products, orders, inventory, payments. Multi-table transactions, inventory decrement atomicity, and analytical queries over order history.
273 A chat system database Messages, conversations, users. High write throughput, time-ordered queries, and the choice between normalized and denormalized message storage.
274 An analytics pipeline database Star schema with fact and dimension tables. Batch ETL, materialized views, and columnar storage for fast aggregations.
275 A time-series metrics database Timestamp-keyed data with compaction and downsampling. Teaches write-optimized storage and time-based partitioning.
276 A search index as a database system Inverted index, tokenization, ranking. How full-text search engines (Elasticsearch) differ from relational databases.
277 A multi-tenant SaaS schema Shared database with tenant isolation. Row-level security, connection pooling, and preventing cross-tenant data leaks.
278 Mistakes and redesigns Real examples of bad schemas and their fixes: premature optimization, over-normalization, missing indexes, wrong data types.
279 Lessons from each system Each case study reinforces that data modeling is about tradeoffs — no schema is perfect, but some are definitely wrong.
Reveal: How do these ideas connect to the next chapter?
Chapter 28 gave you concrete practice applying everything through case studies. Chapter 29 closes the book — and the journey — by confronting the myths that persist even among experienced developers. You will learn that SQL is not inherently slow, that NoSQL databases absolutely have schemas, that relational databases can scale horizontally, and that every database choice is a set of tradeoffs, not a one-size-fits-all answer. After 28 chapters of building knowledge, this final chapter sharpens your critical thinking.
